## Systemd 服务配置

文件位置: `/etc/systemd/system/geneflow.service`

```ini
[Unit]
Description=GeneFlow Dynamic Website (FastAPI)
After=network.target

[Service]
User=ubuntu
Group=ubuntu
WorkingDirectory=/home/ubuntu/web_app
Environment="PATH=/home/ubuntu/miniconda3/envs/dynamic_web/bin"
Environment="KNOWLEDGE_BASE_DIR=/home/ubuntu/web_app/个人知识网站"
ExecStart=/home/ubuntu/miniconda3/envs/dynamic_web/bin/uvicorn app.main:app --host 127.0.0.1 --port 8000
Restart=always
RestartSec=3

[Install]
WantedBy=multi-user.target
```

### 关键路径
- Python 环境: `/home/ubuntu/miniconda3/envs/dynamic_web/`
- uvicorn 可执行文件: `/home/ubuntu/miniconda3/envs/dynamic_web/bin/uvicorn`
- Nginx 配置: `/etc/nginx/sites-enabled/geneflow`
- 留言数据库: `/home/ubuntu/web_app/comments.db`（每次部署保留）

## main.py 路径解析逻辑

```
KNOWLEDGE_BASE_DIR = 环境变量 → 未设则默认 <项目根>/个人知识网站
DB_PATH           = 环境变量 → 未设则默认 <项目根>/comments.db
TEMPLATES_DIR     = <项目根>/templates（固定）
```

服务器上使用 `KNOWLEDGE_BASE_DIR` 环境变量覆盖，本地则不设环境变量自动走默认。

## 服务器常用命令参考

### 部署脚本 `scripts/deploy.sh`

```bash
#!/usr/bin/env bash
set -euo pipefail

SERVER="${1:-ubuntu@www.geneflow.online}"
REMOTE_DIR="/home/ubuntu/web_app"
LOCAL_DIR="$(cd "$(dirname "$0")/.." && pwd)"

rsync -avz --delete \
    --exclude '.git/' \
    --exclude '.env' \
    --exclude '__pycache__/' \
    --exclude '*.pyc' \
    --exclude '.DS_Store' \
    --exclude '*.db' \
    "$LOCAL_DIR/" "$SERVER:$REMOTE_DIR/"

ssh "$SERVER" "cd $REMOTE_DIR && pip install -r requirements.txt --quiet"
ssh "$SERVER" "sudo systemctl restart geneflow"
```

### Nginx
- 配置文件: `/etc/nginx/sites-enabled/geneflow`
- 语法检查: `sudo nginx -t`
- 重载: `sudo nginx -s reload`
